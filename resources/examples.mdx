Design OS is built for designing product interfaces before implementation. Here are common use cases and project types where Design OS fits naturally into your workflow.

## SaaS Applications

<CardGroup cols={2}>
  <Card title="Dashboard Tools" icon="chart-line">

  </Card>
  
  <Card title="Workflow Tools" icon="list-check">
    Project management, CRM, task tracking, team collaboration tools. Define user flows for creating, editing, organizing, and completing work.
  </Card>
  
  <Card title="Content Management" icon="folder-open">
    CMSs, documentation tools, knowledge bases. Design interfaces for creating, organizing, publishing, and managing content structures.
  </Card>
  
  <Card title="Data Management" icon="database">

  </Card>
</CardGroup>

## Common Section Patterns

Use Design OS to design these common section types:

<AccordionGroup>
  <Accordion title="List + Detail Views">
    **Examples:** Customer list → Customer detail, Invoice list → Invoice detail, Product catalog → Product page
    
    Design the list view with filtering and sorting, then the detail view showing full record information.
  </Accordion>
  
  <Accordion title="Form Workflows">
    **Examples:** Multi-step onboarding, order checkout, account setup, survey builders
    
    Design each step of the flow, including validation, progress indicators, and navigation between steps.
  </Accordion>
  
  <Accordion title="Data Entry & Editing">
    **Examples:** Create new record, edit existing record, bulk import, inline editing
    
    Design form layouts, field types, validation states, and save/cancel patterns.
  </Accordion>
  
  <Accordion title="Dashboard & Analytics">
    **Examples:** Overview dashboard, reporting screens, metrics visualization, performance tracking
    
    Design charts, KPI cards, date range pickers, and drill-down interactions.
  </Accordion>
  
  <Accordion title="Settings & Configuration">
    **Examples:** User preferences, team settings, billing management, API configuration
    
    Design tabbed layouts, toggle switches, nested settings groups, and confirmation dialogs.
  </Accordion>
  
  <Accordion title="Content Creation">
    **Examples:** Blog post editor, page builder, email composer, design canvas
    
    Design editor interfaces, toolbars, preview modes, auto-save indicators.
  </Accordion>
</AccordionGroup>

## Team Workflows

<CardGroup cols={2}>
  <Card title="Solo Builder" icon="user">
    You're building solo and want to think through your product design before jumping into implementation. Use Design OS to plan, design, then switch to implementation mode with a clear handoff.
  </Card>
  
  <Card title="Designer + Developer" icon="users">
    Designer uses Design OS to create the product vision, design system, and screen designs. Developer receives the export package and implements using the prompts and components.
  </Card>
  
  <Card title="Product Manager + Team" icon="people-group">
    PM uses Design OS to specify product requirements and user flows. Exports to development team with complete specs, components, and test instructions.
  </Card>
  
  <Card title="Agency + Client" icon="handshake">
    Agency uses Design OS for discovery and design phase. Client reviews designs in the browser. Export package serves as the implementation handoff.
  </Card>
</CardGroup>

## When to Use Design OS

<Card title="Good Fit" icon="check" color="#10b981">
  - Building CRUD interfaces and data management UIs
  - Multi-section products with consistent navigation
  - Projects where you want to separate design from implementation
  - Working with AI coding agents and want better prompts
  - Need to design first, then hand off to developers
  - Want production-ready React components as output
</Card>

<Card title="Not Ideal For" icon="xmark" color="#ef4444">
  - Marketing websites or landing pages (too simple)
  - Highly interactive games or canvas-based UIs
  - Non-React projects (components export as React only)
  - Projects that need design-to-code sync (export is one-way)
  - Teams that prefer traditional design tools (Figma, Sketch)
</Card>
